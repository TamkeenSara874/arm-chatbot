System Map: the frontend only ever talks to the API layer, which routes a chat question into the Query Pipeline and a file/review upload into the Ingest Worker. Both of those read and write the same three stores (Postgres, Qdrant, Redis) — that shared storage is how a review ingested a moment ago is immediately answerable, and how a correction or cached answer is visible across every request. The Query Pipeline is the only thing that talks to the paid LLM providers; the Ingest Worker calls a cheap model too (for pulling out dish names), but never the expensive one.
